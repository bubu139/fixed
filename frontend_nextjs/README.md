# MindView Canvas Frontend

This package contains the Next.js frontend for MindView Canvas. After replacing Firebase with Supabase, the
application requires a few environment variables to be present before it can boot locally or in CI.

## Environment configuration

1. Duplicate the [`.env.example`](.env.example) file in this directory and rename the copy to `.env.local`.
2. Fill in the Supabase credentials for your project:
   - `NEXT_PUBLIC_SUPABASE_URL` – the base URL of your Supabase instance.
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` – the public (anon) API key generated by Supabase.
3. Update the optional values if you use different API endpoints or enable Gemini-powered features.

Once the variables are configured, run the development server from the `frontend_nextjs` directory:

```bash
npm install
npm run dev
```

The Supabase provider in `src/supabase/index.tsx` will surface a localized error message if any required variables are
missing at runtime, helping you verify the configuration quickly.
